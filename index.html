PDA Definition

About Pushdown Automata

A Pushdown Automaton (PDA) is a finite automaton augmented with an unbounded stack. It can recognise a strictly larger class of languages than regular finite automata — specifically, the context-free languages.

How it works

At each step the PDA reads one symbol from the input tape (or takes an ε-move reading nothing), peeks at the top of the stack, and — based on the current state, that input symbol, and that stack symbol — transitions to a new state while replacing the stack top with zero or more new symbols.

Transition format
state, inputSymbol, stackTop → nextState, stackPush
  • inputSymbol = ε — consume no input (ε-move)
  • stackTop = ε — match any stack top without popping
  • stackPush = ε — pop the stack top, push nothing
  • stackPush = AZ — push A on top, Z below (rightmost = deepest)
  • Z — the initial bottom-of-stack marker
Acceptance conditions
  • Final state — input fully consumed and machine in an accept state (double-circled in the graph).
  • Empty stack — input fully consumed and stack is completely empty. Accept states are ignored; watch for the stack to drain to nothing.

Both conditions define the same family of languages, but differ in which PDAs recognise which strings.

DPDA vs NPDA
  • DPDA — at most one valid transition per (state, symbol, stackTop). The simulator warns you if your transition table is ambiguous.
  • NPDA — multiple transitions can share the same left-hand side. Every possibility is explored in parallel; the string is accepted if any branch succeeds. Some languages (e.g. even-length palindromes) require NPDA.
Built-in examples
  • aⁿbⁿ — equal number of a's followed by b's. Deterministic; a classic DPDA language.
  • Even palindromes — even-length strings that read the same forwards and backwards. NPDA only.
